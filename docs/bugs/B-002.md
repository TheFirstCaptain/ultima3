# B-002: MISC Table Resource Lengths Differ From Legacy Copy Loops

## Summary

The bundled `MainResources.rsrc` contains `MISC` resources 401, 402, and 403 with 11 bytes each, but the legacy `GetMiscStuff` and `PutMiscStuff` loops copy 12 bytes for the corresponding type-initial, weapon-use, and armour-use tables.

## Status

Open as of 2026-06-05.

This does not block F-012 because fixture extraction preserves the actual resource byte lengths. It should be resolved or intentionally modeled before a modern persistence adapter normalizes mutable `MISC` table sizes.

## Observed Behavior

F-012's read-only parser validates these bundled resource lengths:

- `MISC` 401: 11 bytes, "Type Initial Table"
- `MISC` 402: 11 bytes, "Weapon Use By Class"
- `MISC` 403: 11 bytes, "Armour Use By Class"

Legacy source in `Sources/UltimaMisc.c` copies 12 bytes for each of those table families in both `GetMiscStuff` and `PutMiscStuff`.

## Expected Behavior

Future fixture and persistence work should preserve the legacy observable behavior deliberately. It should not silently expand, truncate, or normalize these records without a compatibility decision.

## Reproduction

1. Run `make -C harness test`.
2. Inspect `harness/resource_fixture_tests.c` for the asserted 11-byte `MISC` record lengths.
3. Inspect `Sources/UltimaMisc.c` in `GetMiscStuff` and `PutMiscStuff` for the 12-byte copy loops.

## Affected Systems

- Legacy source files: `Sources/UltimaMisc.c`
- Portable core: resource parser records exact resource lengths
- Modernization harness: `harness/resource_fixture_tests.c`
- Resources/assets: `Resources/English.lproj/MainResources.rsrc`
- Future work: persistence adapter design and mutable table import/export

## Validation Plan

- Characterize whether the 12th byte has any observable runtime value in the legacy app or save file.
- If a modern adapter preserves the 11-byte resource records, document the adapter behavior explicitly.
- If a modern adapter expands the tables to 12 bytes, document the compatibility rationale and add fixture coverage.
