# B-002: MISC Table Resource Lengths Differ From Legacy Copy Loops

## Summary

The bundled `MainResources.rsrc` contains `MISC` resources 401, 402, and 403 with 11 bytes each, but the legacy `GetMiscStuff` and `PutMiscStuff` loops copy 12 bytes for the corresponding type-initial, weapon-use, and armour-use tables.

## Status

Resolved on 2026-06-10.

The modern native save policy preserves the actual resource record lengths for `MISC` 501, 502, and 503 as 11 bytes. The 12-byte legacy copy loops remain documented as legacy implementation behavior, but the native adapter does not synthesize or accept an extra compatibility byte unless a future real mutable legacy roster fixture proves one is required.

## Observed Behavior

F-012's read-only parser validates these bundled resource lengths:

- `MISC` 401: 11 bytes, "Type Initial Table"
- `MISC` 402: 11 bytes, "Weapon Use By Class"
- `MISC` 403: 11 bytes, "Armour Use By Class"

Legacy source in `Sources/UltimaMisc.c` copies 12 bytes for each of those table families in both `GetMiscStuff` and `PutMiscStuff`.

## Expected Behavior

Fixture and persistence work should preserve the actual resource record lengths deliberately. Native save-domain documents require mutable `MISC` lengths of `16, 11, 11, 11, 64, 16` for records 500 through 505.

## Reproduction

1. Run `make -C harness test`.
2. Inspect `harness/resource_fixture_tests.c` for the asserted 11-byte `MISC` record lengths.
3. Inspect `Sources/UltimaMisc.c` in `GetMiscStuff` and `PutMiscStuff` for the 12-byte copy loops.

## Affected Systems

- Legacy source files: `Sources/UltimaMisc.c`
- Portable core: resource parser records exact resource lengths
- Modernization harness: `harness/resource_fixture_tests.c`
- Resources/assets: `Resources/English.lproj/MainResources.rsrc`
- Future work: persistence adapter design and mutable table import/export

## Validation Plan

- `Core/include/u3_save.h` names the accepted native mutable `MISC` table lengths.
- `u3_save_load_domain_state` rejects save-domain documents whose required `MISC` records use different lengths, including a synthesized 12-byte `MISC` 501 record.
- `make -C harness test` validates default resource lengths, native fixture lengths, and the expanded-record rejection path.

## Residual Risk

No real user-created classic `Ultima III Roster` fixture has been inspected. If one later proves that Resource Manager writes expand the mutable resources to 12 bytes on disk, a new compatibility feature should add an explicit import policy instead of silently changing native save-domain lengths.
