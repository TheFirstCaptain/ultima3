# F-031: Save-Backed Overworld Movement

## Objective

Expand F-025 from an 11x11 bounded movement smoke to save-backed overworld movement using the party position bytes and the full current Sosaria map record.

## User Value

The modern shell should start behaving like a game session: movement changes durable party state and the rendered viewport follows that state instead of a fixed smoke coordinate.

## Non-Goals

- Do not implement town, dungeon, castle, Ambrosia, or combat transitions in this feature.
- Do not implement all terrain side effects or encounters.
- Do not replace the renderer with the final tileset pipeline.
- Do not bypass the AppKit-owned input path or portable core boundary.

## Dependencies

- F-025 provides the first bounded movement and redraw path.
- F-029 provides a meaningful active party.
- F-030 provides current save state that can be mutated and saved.

## Acceptance Notes

- Overworld movement reads and writes the party's save-domain position bytes.
- The renderer builds a viewport from the current Sosaria map around or from the save-backed party position.
- Arrow keys, keypad movement, and legacy numeric movement commands map to the same portable movement commands.
- Movement result data reports moved, blocked, and current position in a testable way.
- The bounded 11x11 smoke initialization is no longer the only movement path, though it may remain as a fallback fixture test.

## Affected Systems

- Legacy source files: `Sources/UltimaMain.c`, `Sources/UltimaMisc.c`, `Sources/UltimaMacIF.c`
- Portable core: `Core/include/u3_overworld.h`, `Core/src/u3_overworld.c`, possibly `Core/include/u3_save.h`
- Modern shell: `ModernShell/Sources/ShellInputAdapter.swift`, `ModernShell/Sources/ShellResourceAdapter.swift`, `ModernShell/Sources/AppDelegate.swift`
- Modernization harness: `harness/overworld_movement_tests.c`, `harness/input_adapter_tests.c`
- Documentation: `ModernShell/README.md`, `docs/features/F-025.md`

## Validation Plan

Add harness tests for position-byte initialization, cardinal movement, edge/wrap behavior once chosen, viewport extraction, legacy letter command mapping, and save-domain position mutation. Add Swift tests for tick-driven movement updating shell state and render frame from current save data.

Run:

```bash
make -C harness test
swift test
rg -n "AppKit|SwiftUI|Metal|AVFoundation|Foundation|SDL|NS[A-Z][A-Za-z]+\\b|NSURL|Sandbox" Core
```

Manual validation:

1. Run `swift run Ultima3ModernShell`.
2. Load or create a current save with an active party.
3. Move with arrow keys and keypad keys.
4. Move with supported legacy letter commands.
5. Confirm the viewport and position status update consistently.
6. Save, relaunch, and confirm the party position persists after F-030.

## Open Questions

- Resolved for this slice: full-map save-backed movement wraps at Sosaria edges, matching the characterized `MapConstrain` behavior used by legacy overworld movement.
- Resolved for this slice: the save-backed viewport is centered on the party; the older top-left smoke frame remains available as a fallback/test helper.
- Resolved for this slice: legacy numeric `8/2/4/6` movement is accepted through the keyboard input adapter; diagonal numeric movement remains future work.
