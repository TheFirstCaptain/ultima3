# F-019: Audio Adapter Spike

## Objective

Define the modern audio adapter boundary that replaces legacy QuickTime and Sound Manager assumptions.

## User Value

Sound and music need to work in the modern shell without coupling game rules to AVFoundation, AppKit, SDL, or any other platform audio implementation.

## Non-Goals

- Do not convert every sound or music asset.
- Do not choose final audio mixing, streaming, or format-conversion policy unless needed for the first adapter.
- Do not add audio framework dependencies to `Core/`.
- Do not remove legacy audio files or legacy source paths.

## Acceptance Notes

- Define a core-facing sound/music event contract.
- Identify the first modern macOS playback backend to test, likely shell-owned and adapter-wrapped.
- Preserve room for AVFoundation or another backend without changing core APIs.
- Document how missing or unsupported legacy audio assets should fail during early shell testing.

## Validation Plan

Validate with adapter tests where possible and a shell/manual playback check when a modern shell target exists. Record skipped playback validation if local tooling or converted fixtures are unavailable.
