# F-019: Audio Adapter Spike

## Objective

Define the modern audio adapter boundary that replaces legacy QuickTime and Sound Manager assumptions.

## User Value

Sound and music need to work in the modern shell without coupling game rules to AVFoundation, AppKit, SDL, or any other platform audio implementation.

## Non-Goals

- Do not convert every sound or music asset.
- Do not choose final audio mixing, streaming, or format-conversion policy unless needed for the first adapter.
- Do not add audio framework dependencies to `Core/`.
- Do not remove legacy audio files or legacy source paths.

## Acceptance Notes

- Added `Core/include/u3_audio.h` and `Core/src/u3_audio.c` as a portable audio event queue using plain C data only.
- The contract supports sound playback, music playback, music stop, sound volume, and music volume events without exposing AVFoundation, AppKit, SDL, or filesystem paths to `Core/`.
- First shell playback backend is `ShellAudioAdapter`, an AVFoundation-backed adapter owned by `ModernShell`.
- The first shell smoke path is `Game > Test Sound`, which enqueues `U3_AUDIO_SOUND_ERROR1` and resolves `Resources/Sounds/Error1.wav`.
- Missing or unsupported files fail visibly in shell status text as `Audio Missing <name>` or `Audio Failed <name>` rather than crashing or changing core behavior.
- Music event support is present in the portable contract and shell adapter, but HIL music playback is deferred until a later resource/persistence or audio asset validation slice.

## Affected Systems

- Legacy source files: reference only, primarily `Sources/UltimaSound.m`, `Sources/CocoaBridge.m`, and `Sources/LWSoundPlayer.m`.
- Portable core: added `u3_audio` queue/event contract.
- Modern shell: added `ShellAudioAdapter`; added `Game > Test Sound` smoke command.
- Modernization harness: added `harness/audio_adapter_tests.c` and integrated it into `harness/Makefile`.
- Resources/assets: read-only use of existing `Resources/Sounds/Error1.wav` for manual shell validation.
- Documentation: updated this feature record, the feature tracker, harness README, and modern shell README.
