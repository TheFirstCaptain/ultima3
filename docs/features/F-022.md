# F-022: Resource-Backed Render Smoke

## Objective

Render a smoke frame using real bundled resource-derived data instead of the synthetic tile grid.

## User Value

The modern shell should prove that resource loading and rendering can work together before adding playable state transitions.

## Non-Goals

- Do not convert every legacy graphic, map, or animation resource.
- Do not replace the renderer adapter contract unless F-020 or F-017 findings require it.
- Do not move resource parsing into the AppKit view.
- Do not implement free movement or game-loop orchestration.

## Dependencies

- F-020B should prove bundle resource adapter reads.
- F-017 remains the renderer contract baseline.
- F-015 provides representative map, talk, dungeon, and combat fixture expectations.

## Acceptance Notes

- The rendered smoke frame is derived from bundled or fixture-backed resource data.
- Resource parsing remains adapter-owned and does not leak platform types into `Core/`.
- The shell can display the frame through the existing render adapter path.
- Manual HIL verification confirms that the frame is visible, stable, and not using the old synthetic placeholder grid.

## Open Questions

- Resolved for first implementation: use `CONS` 400 as the first resource-backed smoke target because its first 121 bytes match the F-017 11x11 render grid contract and are characterized by F-015.
- Resolved for first implementation: continue reading `Resources/English.lproj/MainResources.rsrc` through `ShellResourceAdapter`; do not introduce converted assets in this slice.
