# F-059: Combat Spell Coverage Tranche

## Objective

Expand the current first Mittar spell path into a bounded combat spell set with shared prompt, cost, targeting, result, mutation, and feedback rules.

## User Value

Combat becomes a meaningful party-class system instead of a mostly physical attack loop. This also forces the shell/core contract to handle multiple spell shapes before non-combat magic is attempted.

## Non-Goals

- Do not implement town, dungeon, or overworld non-combat spells.
- Do not add a final legacy spell-book UI if the existing smoke command and prompt model is enough for this tranche.
- Do not change existing melee, ranged, monster-turn, flee, victory, or defeat rules except where spell outcomes need to feed the same sequencing.
- Do not make combat state durable.

## Dependencies

- F-050 provides combat command routing and direction prompts.
- F-051 provides post-player monster-turn sequencing.
- F-054 provides the first combat spell command and magic-spend write-back.
- F-058 may provide reusable presentation feedback for spell effects.

## Acceptance Notes

- The portable combat command contract supports a selected bounded set of additional combat spells with explicit unsupported statuses for the remaining spell list.
- Spell eligibility, magic cost, target selection, damage/heal/status mutation, audio, and redraw behavior are deterministic under supplied rolls.
- Roster magic and any HP/status mutations write to the current in-memory native document and mark it unsaved, while the active combat session remains transient.
- Spell results either consume a handled player turn and run existing monster sequencing, or explicitly report why no turn was consumed.
- Existing Mittar, melee, ranged, flee, victory, and defeat tests remain stable.

## Affected Systems

- Legacy source files: combat spell command and damage/status helpers in `Sources/UltimaMain.c`, `Sources/UltimaMisc.c`, and related combat modules
- Portable core: combat player command/spell result contract
- Modern shell: combat spell command routing, prompt handling, status/audio mapping, and document write-back
- Modernization harness and Swift tests: deterministic spell eligibility, cost, targeting, mutation, and sequencing coverage
- Documentation: `ModernShell/README.md`, feature tracker

## Validation Plan

Cover at least one direct-damage spell, one status-affecting or multi-target spell if safely bounded, invalid caster, insufficient magic, invalid target, defeated target suppression, document mutation, and no-mutation rejection paths. Run `make -C harness test`, `swift test`, and the Core boundary scan.

Manual validation: enter combat, cast supported spells with valid and invalid casters, confirm visible feedback, magic spend, HP/status mutation, monster-turn sequencing, and save rejection until combat exits.

## Open Questions

- Which spell set should be included first after Mittar: lowest-risk wizard combat spells, cleric recovery spells, or one representative of each shape?
- Should the shell expose a temporary spell index prompt, mnemonic commands, or continue one explicit smoke command per supported spell?
- Which spell effects need characterization before extraction rather than direct bounded implementation?

## Completion Notes

Completed on 2026-07-21.

- Added bounded Core support for two additional wizard combat spells:
  - `Fulgar` (`U` in the shell): directed projectile spell, 25 magic, fixed 75 damage, Immolate feedback.
  - `Noxum` (`O` in the shell): non-directional all-live-monster damage spell, 50 magic, fixed 75 damage, BigDeath feedback.
- Preserved `M` as the existing Mittar shortcut and kept the direction prompt model for directed spells.
- Added explicit unsupported handling for spells outside the selected slice.
- Added multi-target spell result metadata, aggregate experience-award support, and document write-back for roster magic/experience mutations.
- Kept combat sessions transient and Save rejected until the party returns to Sosaria.
- The Noxum slice intentionally uses deterministic all-live-monster damage for validation; the full legacy per-monster BigDeath random suppression remains deferred.
