# F-003D: Extract Combat Predicates

## Objective

Move the characterized combat predicate behavior from the harness copy into shared portable core code, then update the harness tests to call that shared implementation.

## User Value

Combat collision and passability rules are a useful early game-core boundary. Extracting them after map/math and strings proves the portable core can handle small stateful rule modules.

## Non-Goals

- Do not port the full combat loop.
- Do not include `ValidMove` unless audio side effects are isolated.
- Do not port monster AI, rendering, projectiles, victory handling, or character damage.

## Acceptance Notes

- Shared portable implementation covers `CombatValidMove`, `CombatMonsterHere`, and `CombatCharacterHere`, or namespaced equivalents.
- Harness tests no longer carry local copied implementations for these functions.
- Passability, monster lookup priority, character lookup priority, and inactive-coordinate behavior remain covered.
- Combat predicate state is explicit enough to avoid depending on legacy app globals.

## Affected Systems

- Legacy source files: reference only
- Modernization harness: `harness/combat_predicates_tests.c`, `harness/Makefile`
- Resources/assets: none
- Build or tooling: portable core source and header paths
- Documentation: validation strategy and possible architecture notes

Run:

```bash
make -C harness test
```
