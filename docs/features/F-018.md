# F-018: Input Adapter Spike

## Objective

Map AppKit keyboard, mouse, menu, and macro inputs into portable core-facing command events.

## User Value

Ultima III depends on dense keyboard commands and Mac command behavior. A clear input adapter prevents AppKit event handling from leaking into the portable core and preserves command fidelity.

## Non-Goals

- Do not rewrite all legacy input handling in one pass.
- Do not add AppKit types to `Core/`.
- Do not change characterized core command ordering such as autocombat macro execution.
- Do not finalize every mouse/cursor interaction before the shell host exists.

## Acceptance Notes

- Added `Core/include/u3_input.h` and `Core/src/u3_input.c` as a portable input event queue using plain C data only.
- The queue supports keyboard key events, mouse-down events, menu command events, and macro key events without exposing AppKit types to `Core/`.
- Macro events are appended and consumed in execution order, preserving the accepted `u3_autocombat_macro.commands[0]` first-consumed ordering from `DECISION_LOG.md`.
- The first shell smoke commands are direct keyboard input, primary mouse-down coordinates, and menu commands for New Game and Save.
- `ShellInputAdapter` translates AppKit-owned shell events into the portable queue, then consumes the same queue to report the shell smoke status. Menu commands and direct key commands therefore converge before the shell presents the consumed command.

## Affected Systems

- Legacy source files: reference only, primarily `Sources/UltimaMacIF.c`, `Sources/UltimaMain.h`, and `Sources/UltimaAutocombat.c`.
- Portable core: added `u3_input` queue/event contract.
- Modern shell: added `ShellInputAdapter`; updated `GameHostView` and menu actions to route through the input queue.
- Modernization harness: added `harness/input_adapter_tests.c` and integrated it into `harness/Makefile`.
- Resources/assets: none.
- Documentation: updated this feature record, the feature tracker, harness README, and modern shell README.
