# F-013: Resource Data Extraction Inventory

## Objective

Enumerate the records in `Resources/English.lproj/MainResources.rsrc` and map each runtime-relevant resource type/id to its legacy consumer and likely converted-data path.

## User Value

The modern port cannot remove `MainResources.rsrc` until the project knows exactly which records are still required. This feature turns the opaque resource archive into an actionable conversion and adapter backlog.

## Non-Goals

- Do not convert all resources to a new format.
- Do not remove the `PBXRezBuildPhase`.
- Do not change bundled assets or source behavior.
- Do not decide final renderer, audio, or persistence formats.

## Acceptance Notes

- Enumerate resource records for known runtime families such as `MAPS`, `MONS`, `TLKS`, `MISC`, `PRTY`, `ROST`, `CONS`, `DEMO`, `SGNT`, `PICT`, `snd `, and legacy string/dialog resources.
- For each runtime-relevant family, identify source consumers and whether the data appears read-only, template data, or save-related.
- Document the toolchain or parser used, including failure modes if legacy tools cannot read the archive.
- Recommend follow-up extraction or characterization targets based on resource risk and modernization value.

## Result

Status: complete as of 2026-06-05.

F-013 extended the F-012 classic resource parser with iteration APIs:

- `u3_resource_get_type`
- `u3_resource_get_record`

`harness/resource_fixture_tests.c` now validates that `Resources/English.lproj/MainResources.rsrc` contains 45 resource types and 303 total records, and asserts runtime-relevant family counts for `MAPS`, `MONS`, `TLKS`, `CONS`, `MISC`, `STR#`, and `PICT`.

The archive is a classic data-fork resource file with:

- data offset: 256
- map offset: 180264
- data length: 180008
- map length: 5621

The previous local `DeRez` failure remains a legacy-tooling fact from F-010, but it no longer blocks inventory work because the checked-in archive is parseable directly by the project parser.

## Recommended Follow-Ups

1. `F-015: Map/Talk/Combat Fixture Characterization`
   Use the enumerated `MAPS`, `MONS`, `TLKS`, and `CONS` records to pick representative fixture IDs. Good first candidates are one town/castle map in 400..411, one dungeon map in 412..418, Ambrosia 421, and one 200-byte `CONS` combat fixture plus one 121-byte special fixture.

2. Resource extraction tool hardening
   The parser can enumerate and return byte slices, but there is no command-line extractor yet. Add one only when fixture files need to be generated or checked into a stable location.

3. Legacy string/UI resource cleanup
   Before removing `MainResources.rsrc`, remaining `GetIndString`, `Alert`, `GetNewDialog`, `GetMenuHandle`, `GetCursor`, and `GetPicture` paths need a consumer-level replacement plan.

4. Persistence adapter design after fixture characterization
   `F-014` should use F-012 and F-013 evidence, plus any F-015 fixture results, before choosing legacy import/export versus new-format-only behavior.
