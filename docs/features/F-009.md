# F-009: Combat Action Resolution Inventory

## Objective

Map the combat action resolution surface before extracting larger combat behavior.

## User Value

The current portable combat work covers predicates and autocombat decisions, but not the effects of attacks, spells, monster damage, poison, victory, or combat turn progression. This inventory should identify the smallest extractable combat resolution slices and the side effects that must be modeled first.

## Non-Goals

- Do not port the full combat loop in one step.
- Do not implement attack, spell, projectile, poison, victory, or loot resolution during the inventory.
- Do not change existing portable combat APIs unless a narrow documentation correction requires it.
- Do not modify resources or historical assets.

## Acceptance Notes

- Inventory the main combat action functions and side effects in `Sources/UltimaSpellCombat.c`.
- Identify dependencies on globals, rendering, sound, text, random numbers, resources, and party state.
- Recommend the next one to three characterization targets after autocombat and dungeon navigation.
- Capture findings in modernization documentation and, if useful, propose child features for concrete extraction work.
- No code extraction is required for this milestone.

## Result

Status: complete as of 2026-06-02.

Inspected `Sources/UltimaSpellCombat.c`, `Sources/UltimaSpellCombat.h`, `Sources/UltimaIncludes.h`, related call sites, and existing modernization strategy docs. Added combat action resolution findings to `docs/modernization/legacy-inventory.md`.

The main finding is that `Combat()` is too broad to extract directly. It owns combat setup, player input, animation, preference reads, text, sound, autocombat handoff, player turns, monster turns, victory, and restoration of overworld or dungeon state. The next extraction should instead use narrower slices where state transitions can be modeled as explicit core results and UI/audio/text can become events.

Recommended next characterization targets:

- `F-009A`: `DamageMonster` and monster defeat rewards.
- `F-009B`: `CombatAttack` player attack resolution after direction selection.
- `F-009C`: monster attack side effects, including `Poison`, `Pilfer`, hit/miss, movement, shooting, and spell targeting.

Deferred for later inventory or extraction:

- Full `Combat()` loop orchestration.
- Spell selection and `Spell()` as a whole.
- Resource-backed combat screen loading through `GetScreen`.
- Rendering, input, audio, and preference adapters.

## Affected Systems
