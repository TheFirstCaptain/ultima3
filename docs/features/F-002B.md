# F-002B: Map/Math Accessor Characterization

## Objective

Characterize the first recommended legacy behavior slice: map and math accessors in `Sources/UltimaMisc.c`.

## User Value

These functions underpin movement, map wrapping, and later combat logic. Capturing their current behavior gives the port an early, useful safety net.

## Non-Goals

- Do not refactor `UltimaMisc.c`.
- Do not port map rendering or resource loading.
- Do not attempt full save/load compatibility.

## Acceptance Notes

- Characterization covers `GetHeading`, `Absolute`, `GetXY`, `PutXY`, `MapConstrain`, `GetXYVal`, and `PutXYVal`, or documents why any are deferred.
- Tests use synthetic globals/arrays and avoid resource files.
- Edge cases include byte-wrap behavior, map wrapping, and out-of-bounds behavior affected by `Party[3]`.
- Harness command is documented in `docs/modernization/validation-strategy.md`.

## Result

Status: complete as of 2026-05-30.

The initial harness lives in `harness/map_math_accessors_tests.c` and covers:

- `GetHeading`
- `Absolute`
- `GetXY`
- `PutXY`
- `MapConstrain`
- `GetXYVal`
- `PutXYVal`

It uses synthetic `TileArray`, `Party`, `Map`, `gMapOffset`, and `gCurMapSize` state. It covers byte-like wrapping in `Absolute`, 11-column tile indexing, map coordinate wrapping, surface out-of-bounds wrapping, non-surface out-of-bounds masking through `Party[3]`, and wrapped unsigned coordinates in `PutXYVal`.

## Affected Systems

- Legacy source files: `Sources/UltimaMisc.c`, related globals from `Sources/UltimaMain.c`
- Modernization harness: first executable harness/test files
- Resources/assets: none expected
- Build or tooling: based on F-002A
- Documentation: validation strategy, possible decision log updates
