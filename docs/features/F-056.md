# F-056: Dungeon Chest Trap and Equipment Rewards

## Objective

Expand the current dungeon chest interaction beyond bounded gold rewards to cover trap resolution and first weapon/armour reward paths.

## User Value

Dungeon chests should feel less like a placeholder. Trap and equipment outcomes connect dungeon exploration to roster risk and inventory progression.

## Non-Goals

- Do not implement every loot table or shop/equipment UI.
- Do not implement combat loot if F-052 keeps victory rewards narrower.
- Do not make transient dungeon tile clearing durable unless a separate persistence decision requires it.
- Do not add modal blocking prompts beyond the existing non-blocking character selection pattern.

## Dependencies

- F-047 provides prompt-backed chest opening, transient tile clearing, and gold reward mutation.
- F-051/F-053 may provide reusable inventory write-back patterns for weapon and armour bytes.
- F-036 defines manual save durability.

## Acceptance Notes

- Chest trap branches return explicit portable results and apply bounded HP/status mutations to the selected or affected active party member.
- First weapon and armour reward branches mutate active roster inventory bytes in the current in-memory native document.
- Existing gold reward behavior remains stable.
- Invalid chest state and unsupported interaction paths report explicit status without corrupting session or document state.
- Save remains rejected while in dungeon mode; mutations become durable only after returning to Sosaria and saving.

## Affected Systems

- Legacy source files: `Sources/UltimaMain.c` `GetChest`, `Sources/UltimaDngn.c` chest interaction paths
- Portable core: dungeon chest reward/trap contracts
- Modern shell: chest status/audio/redraw and document mutation handling
- Save adapter: roster HP/status/gold/weapon/armour inventory mutation
- Modernization harness and Swift tests: trap damage/status, disarm, weapon, armour, gold, invalid/no-mutation cases
- Documentation: `ModernShell/README.md`, feature tracker

## Validation

- `make -C harness test`
- `swift test`
- `rg -n "AppKit|SwiftUI|Metal|AVFoundation|Foundation|SDL|NS[A-Z][A-Za-z]+\\b|NSURL|Sandbox" Core`

Manual validation: open dungeon chests with deterministic trap and reward cases, inspect party/roster status, return to Sosaria, save, reload, and confirm durable intended mutations.

## Resolution

- Implemented selected-character chest trap handling for disarm, bounded damage, and poison status.
- Implemented the first deterministic weapon and armour inventory reward branches while preserving bounded gold rewards.
- Chest tile clearing remains transient to the dungeon session; roster gold, HP/status, weapon inventory, and armour inventory mutations remain in the current in-memory native document until the party returns to Sosaria and saves.
