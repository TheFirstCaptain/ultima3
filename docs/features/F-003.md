# F-003: Portable Core Seed

## Objective

Create the first shared portable game-core code from behavior already characterized by the harness, starting with small deterministic functions and keeping the legacy source as the reference implementation.

## User Value

This converts the harness from copied behavior checks into a real porting safety net. Once the first portable core module exists, future modernization work can move behavior out of legacy platform files in small, testable increments.

## Non-Goals

- Do not port the full game loop.
- Do not create the modern macOS app shell.
- Do not replace rendering, audio, input, resource loading, or persistence.
- Do not refactor broad legacy source files.
- Do not change historical assets.

## Acceptance Notes

- F-003A defines the initial portable core layout and naming convention.
- F-003B moves map/math accessor behavior into shared portable code used by the harness.
- F-003C moves Pascal string helper behavior into shared portable code used by the harness.
- F-003D moves combat predicate behavior into shared portable code used by the harness.
- Harness tests continue to pass with `make -C harness test`.
- Any durable architecture or tooling decision is recorded in `DECISION_LOG.md`.

## Result

Status: complete as of 2026-05-30.

The first shared portable core modules now live under:

```text
Core/
  include/
    u3_combat.h
    u3_map_math.h
    u3_pascal_string.h
  src/
    u3_combat.c
    u3_map_math.c
    u3_pascal_string.c
```

The harness tests now compile those shared implementations instead of carrying local copied implementations.

Completed child features:

- F-003A: accepted top-level `Core/` layout with lowercase `u3_` APIs and explicit caller-owned state guidance.
- F-003B: extracted map/math accessors to `u3_map_math`.
- F-003C: extracted Pascal string helpers to `u3_pascal_string`.
- F-003D: extracted combat predicates to `u3_combat`.
