# F-003C: Extract Pascal String Helpers

## Objective

Move the characterized Pascal string helper behavior from the harness copy into shared portable core code, then update the harness tests to call that shared implementation.

## User Value

Text and resource strings are central to the game. Extracting byte-level string helpers creates a reusable compatibility layer before modern string conversion work begins.

## Non-Goals

- Do not modernize text rendering.
- Do not convert string resources.
- Do not change the quirks captured in F-002C.

## Acceptance Notes

- Shared portable implementation covers `u3_string_location`, `u3_search_replace`, `u3_is_newline`, and `u3_add_string`.
- The public header uses repo-owned `u3_pascal_string` rather than Classic Mac `Str255`.
- Harness tests no longer carry local copied implementations for these functions.
- Terminal-match and length-byte quirks remain covered.
- Move/copy shims are isolated inside the portable module and use standard `memmove`.

Status: complete as of 2026-05-30.
