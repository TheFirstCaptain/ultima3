# F-007: Autocombat Macro Decision Core

## Objective

Characterize and extract the full `AutoCombat` macro decision flow into the portable core.

## User Value

F-004 through F-006 extracted the small autocombat helpers for targeting, movement forecasting, danger checks, and low-health checks. The next useful boundary is the top-level decision function: given combat state and character state, choose the same macro key sequence the legacy `AutoCombat` function would enqueue.

This creates a portable, testable autocombat command generator without requiring Carbon preferences, input polling, rendering, audio, or the legacy macro buffer.

## Non-Goals

- Do not port rendering, audio, input polling, resource loading, or preferences APIs.
- Do not port spell effect resolution or weapon attack resolution.
- Do not rewrite the full combat loop in `Sources/UltimaSpellCombat.c`.
- Do not modify legacy `Sources/` unless an adapter step is explicitly chosen later.
- Do not change behavior captured from `Sources/UltimaAutocombat.c`.

## Acceptance Notes

- Portable implementation covers the top-level `AutoCombat(chnum)` decision flow, or a namespaced equivalent.
- The portable API returns a bounded macro command sequence instead of calling `AddMacro`.
- Required legacy globals become explicit state, including character class, magic, weapon, HP, alive state, monster type, spell-cast flags, diagonal preference, and combat positions.
- Harness tests cover the major decision branches:
  - low-HP flee movement
  - Repond against orcs
  - Pontori against skeletons
  - high-threat wizard spell
  - high-threat cleric spell
  - Sanctu healing target selection
  - Mittar or projectile lined-up attack
  - projectile line-up movement
  - melee adjacent attack
  - melee nearest-monster movement
  - nearly-dead melee pass behavior
- `make -C harness test` passes.

## Result

Status: complete as of 2026-05-31.

Extended:

- `Core/include/u3_autocombat.h`
- `Core/src/u3_autocombat.c`
- `harness/autocombat_targeting_tests.c`

Added `u3_autocombat_auto_combat`, which returns a bounded `u3_autocombat_macro` command sequence instead of writing into the legacy macro buffer with `AddMacro`. The returned commands are in legacy execution order, matching `GetKeyMouse` consumption of `Macro[0]`, not raw `AddMacro` call order.

Expanded `u3_autocombat_state` with explicit character class, magic, weapon, career table, and spell-cast flag fields formerly read through `Player`, `Party`, `careerTable`, `g5521`, and `g56E7`.

The API remains in `u3_autocombat` for now. Macro capacity is `U3_AUTOCOMBAT_MACRO_CAPACITY`, currently 8 bytes, which covers the characterized legacy branches while leaving room for adapter use. Raw legacy byte values remain in use at the extraction boundary per the portable core extraction rules.
