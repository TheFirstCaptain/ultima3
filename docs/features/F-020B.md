# F-020B: Bundle Resource Adapter Smoke

## Objective

Use a shell-provided resource root to read and validate one known bundled resource fixture through an adapter boundary.

## User Value

Resource adapter integration should prove that modern shell path ownership can feed existing parser/fixture logic without reviving Resource Manager dependencies.

## Non-Goals

- Do not convert all resources.
- Do not remove `MainResources.rsrc`.
- Do not write save data.
- Do not put filesystem or AppKit APIs in `Core/`.

## Acceptance Notes

- Adapter can receive a shell-provided resource root or file path as portable data.
- Adapter can open `Resources/English.lproj/MainResources.rsrc` and validate at least one known fixture from F-012/F-013.
- Missing or malformed resource input reports an adapter error rather than partial state.
- Existing resource parser remains portable and independent of shell APIs.

## Affected Systems

- Legacy source files: reference only, primarily resource consumers documented by F-013.
- Modern shell: resource root handoff.
- Modernization harness: fixture-backed resource adapter smoke test.
- Resources/assets: read-only use of `MainResources.rsrc`.
- Documentation: feature doc and tracker.

## Validation Plan

Run `make -C harness test`, build the shell if shell status is added, run the Core boundary scan, and optionally perform HIL validation that the shell reports resource adapter success/failure.

Validation passed on 2026-06-06 with:

```bash
swift build --product Ultima3ModernShell
make -C harness test
rg -n "AppKit|SwiftUI|Metal|AVFoundation|Foundation|SDL|NS[A-Z][A-Za-z]+\\b|NSURL|Sandbox" Core
```

The boundary scan produced no matches.

Manual shell validation is still recommended by running `swift run Ultima3ModernShell` from the repository root and confirming the status overlay reports `Resource OK MainResources.rsrc`.

## Open Questions

- Resolved for this smoke slice: the adapter reads `Resources/English.lproj/MainResources.rsrc` directly for now.
- Deferred: generated fixture files may still be useful once converted resources become source-of-truth assets.

## Status

Complete as of 2026-06-06.

Added `ShellResourceAdapter`, which receives the shell-provided resource root as a plain string, reads `MainResources.rsrc` with Foundation inside the shell, and passes raw bytes into the portable `u3_resource` parser. Parser-backed records are validated only within the `Data.withUnsafeBytes` lifetime. The smoke checks the known 45-type inventory plus `PRTY` 500 and `MAPS` 420 fixture lengths. The harness now includes root-path resource validation and missing-root failure smokes that mirror the shell handoff.
