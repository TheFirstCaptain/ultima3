# F-020: Resource and Persistence Shell Integration

## Objective

Wire shell-owned bundle and save-file locations into the resource and persistence adapters defined by earlier modernization work.

## User Value

The modern app shell must provide concrete locations for bundled data and player saves while preserving F-014's adapter boundary and legacy save timing constraints.

## Non-Goals

- Do not change the F-014 persistence format decision.
- Do not promise exact legacy save export.
- Do not make the persistence adapter depend directly on AppKit, sandbox, Preferences-folder, or application-support APIs.
- Do not convert all resources during this milestone.

## Acceptance Notes

- The AppKit shell provides concrete bundle and save-file URLs or paths to adapters.
- Persistence serialization remains adapter-owned and independent of AppKit types.
- Manual save eligibility and transition/auto-save timing remain aligned with F-014.
- Resource loading can use F-012 through F-015 fixtures or converted bundle data without relying on writable Resource Manager files.

## Validation Plan

Use fixture-backed adapter tests for resource and persistence behavior. Add shell-level smoke checks once the AppKit shell can provide real bundle and save locations.
