# F-016: Minimal AppKit/SwiftUI Shell Seed

## Objective

Create the first modern macOS shell target using the F-011 direction: AppKit owns the app lifecycle, windowing, menus, command routing, and game host view; SwiftUI is available only for secondary panels.

## User Value

This gives the portable core and future adapters a real modern host without rewriting the legacy Xcode project or prematurely choosing final rendering, audio, or persistence implementations.

## Non-Goals

- Do not port the full game loop.
- Do not choose the final renderer, audio stack, save format, or asset conversion path.
- Do not remove or rewrite the legacy `Ultima3.xcodeproj`.
- Do not move game rules into AppKit or SwiftUI.

## Acceptance Notes

- Added a SwiftPM scaffold with a `Ultima3ModernShell` executable product and a plain C `Ultima3Core` library target.
- AppKit owns application lifecycle, primary window, menu/command routing, and a placeholder game host view in `ModernShell/Sources/`.
- SwiftUI integration is limited to `ModernShellPreferencesView`, a secondary preferences panel hosted from AppKit.
- The shell links and calls `Core/` through the C target without exposing AppKit or SwiftUI types through core headers.
- The legacy `Ultima3.xcodeproj` remains unchanged.

## Affected Systems

- Legacy source files: none.
- Modernization harness: none beyond existing validation.
- Resources/assets: none.
- Build or tooling: added root `Package.swift` and `ModernShell/`.
- Documentation: added `ModernShell/README.md` and updated this feature record.
