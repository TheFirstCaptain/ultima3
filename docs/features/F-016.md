# F-016: Minimal AppKit/SwiftUI Shell Seed

## Objective

Create the first modern macOS shell target using the F-011 direction: AppKit owns the app lifecycle, windowing, menus, command routing, and game host view; SwiftUI is available only for secondary panels.

## User Value

This gives the portable core and future adapters a real modern host without rewriting the legacy Xcode project or prematurely choosing final rendering, audio, or persistence implementations.

## Non-Goals

- Do not port the full game loop.
- Do not choose the final renderer, audio stack, save format, or asset conversion path.
- Do not remove or rewrite the legacy `Ultima3.xcodeproj`.
- Do not move game rules into AppKit or SwiftUI.

## Acceptance Notes

- Add a modern macOS target or shell scaffold that can build independently of Carbon, QuickTime, and legacy SDK paths.
- AppKit owns application lifecycle, primary window, menu/command routing, and a placeholder game host view.
- SwiftUI integration is limited to a placeholder or boundary for preferences/secondary panels.
- The shell can call or link plain C code without exposing AppKit types through `Core/`.
- Document any build limitations on the current local toolchain.

## Validation Plan

Attempt a modern shell build where the local Xcode/toolchain allows it. If a build is not possible, record the exact blocker and validate the project changes by inspection.
