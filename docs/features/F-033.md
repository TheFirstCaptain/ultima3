# F-033: Overworld Turn Economy Slice

## Objective

Add the first save-backed overworld turn consequences after movement and passability, focused on deterministic per-step state updates such as food or movement counters.

## User Value

Movement should start feeling like actual gameplay instead of free viewport scrolling. A narrow turn-economy slice gives each accepted movement command a visible state consequence while keeping random encounters, transitions, and combat out of scope.

## Non-Goals

- Do not implement full monster movement or encounter generation.
- Do not implement town, dungeon, castle, Ambrosia, or combat transitions.
- Do not implement every legacy hunger, poison, vehicle, or time-of-day side effect in this slice.
- Do not let the shell mutate save bytes directly; portable core or persistence-domain helpers must own rule-level mutations.

## Dependencies

- F-031 should provide save-backed party position and rendering.
- F-032 should provide explicit passable versus blocked movement results.
- F-030 provides current native save ownership, load, and writeback; this feature must add or reuse explicit mutation helpers for the selected state update.

## Acceptance Notes

- Selected first turn-economy family: the legacy `PRTY` move counter in party bytes 11 through 14, represented in core as zero-based offsets 10 through 13.
- Handled overworld movement attempts add active party size to the move counter, matching `Sources/UltimaMisc.c` `IncMoves`.
- Blocked movement consumes the same counter cost as successful movement because the legacy `Game` loop calls `Routine6E35` after movement commands even when `NoGo` is reached.
- The portable move result reports the applied turn delta and before/after move-counter values for shell status and future message/audio adapters.
- The shell reports the changed counter after save-backed movement, and the changed value persists after write/relaunch/load.

## Affected Systems

- Legacy source files: `Sources/UltimaMain.c`, `Sources/UltimaMisc.c`
- Portable core: likely `Core/include/u3_overworld.h`, `Core/src/u3_overworld.c`, and `Core/include/u3_save.h`
- Modern shell: tick-driven movement path and status/inspection path
- Modernization harness: `harness/overworld_movement_tests.c`, `harness/save_fixture_tests.c`
- Documentation: feature tracker, `ModernShell/README.md`, and any durable byte-contract notes

## Open Questions

- Resolved for this slice: movement counter is the first state family; food remains deferred because it uses per-roster hundredths and starvation damage.
- Resolved for this slice: blocked movement consumes a turn, following the legacy post-command `Routine6E35` path.
- Resolved for this slice: the value is exposed in game-host command status for save-backed movement; broader inspectors wait for a later UI/debug feature.
