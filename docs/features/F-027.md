# F-027: MISC Table Length Policy

## Objective

Resolve and document the modern persistence policy for mutable `MISC` tables whose bundled resource lengths differ from the legacy copy loops.

## User Value

Save and resource behavior needs to be predictable before modern gameplay starts mutating durable state. This feature keeps a known compatibility edge from becoming hidden inside later persistence work.

## Non-Goals

- Do not rewrite the persistence format chosen in F-014.
- Do not edit historical bundled resources.
- Do not implement exact legacy Resource Manager export.
- Do not normalize unrelated save-domain records.

## Dependencies

- B-002 tracks the observed mismatch for `MISC` 401, 402, and 403.
- F-012 and F-013 characterize default resource lengths.
- F-014 keeps mutable `MISC` records as explicit byte records.
- F-020 and F-021 provide native save fixture and domain loading paths.

## Acceptance Notes

- The project has a documented policy for `MISC` 501, 502, and 503 lengths in native save documents.
- The policy explicitly explains how the modern adapter treats the legacy 11-byte resource data versus 12-byte copy loops.
- Harness coverage validates accepted fixture lengths and rejects or preserves mismatched lengths according to the policy.
- B-002 is either resolved with the policy or left open with a narrowed follow-up reason.

## Implementation Notes

- Accepted native mutable `MISC` record lengths are named in `Core/include/u3_save.h`: `16, 11, 11, 11, 64, 16` for records 500 through 505.
- `Core/src/u3_save.c` uses the shared length policy for new-game fixture building, legacy roster import sizing/validation, and save-domain loading.
- Native save-domain loading rejects expanded 12-byte variants of `MISC` 501, 502, and 503 instead of adding an implicit compatibility byte.
- B-002 is resolved with residual risk documented there: a future real mutable legacy roster fixture may still justify a separate import compatibility policy.

## Affected Systems

- Legacy source files: `Sources/UltimaMisc.c`
- Portable core: `Core/include/u3_save.h`, `Core/src/u3_save.c`
