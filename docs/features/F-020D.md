# F-020D: Save Path and Atomic Write Smoke

## Objective

Have the shell provide a save-document path and have an adapter atomically write and read a small validated smoke document.

## User Value

This proves the modern save path and atomic write behavior before the adapter writes real party, roster, map, or mutable-table data.

## Non-Goals

- Do not save real gameplay state.
- Do not implement auto-save or transition-save timing.
- Do not import legacy roster files.
- Do not choose final user-facing save-management UI.

## Acceptance Notes

- Shell owns the concrete save path.
- Adapter owns serialization and atomic replacement behavior.
- Adapter can round-trip a small smoke document and reject malformed input.
- Failed writes retain the previous valid document.

## Affected Systems

- Legacy source files: reference only, primarily save timing paths in `Sources/UltimaMain.c` and `Sources/UltimaMisc.c`.
- Modern shell: save path provider and status.
- Modernization harness: adapter write/read tests using temporary paths.
- Resources/assets: none.
- Documentation: feature doc and tracker.

## Validation Plan

Run adapter tests for successful write/read, malformed input rejection, and previous-document retention on failed replacement. Build shell, run Core boundary scan, and perform HIL validation of save-path smoke status if exposed.

Validation passed on 2026-06-06 with:

```bash
swift build --product Ultima3ModernShell
swift test
make -C harness test
rg -n "AppKit|SwiftUI|Metal|AVFoundation|Foundation|SDL|NS[A-Z][A-Za-z]+\\b|NSURL|Sandbox" Core
```

The boundary scan produced no matches.

Manual shell validation is still recommended by running `swift run Ultima3ModernShell`, choosing Game > Write Save Smoke, and confirming the status overlay reports `Save OK Smoke | Save Read OK Smoke`.

## Open Questions

- Resolved for this smoke slice: malformed replacement payloads are the practical failure injection for previous-document retention. The adapter rejects invalid data before replacement, and harness tests confirm the previous valid document remains readable.
- Deferred: lower-level filesystem failure injection can be revisited if F-020D grows into a production persistence backend rather than a smoke adapter.

## Status

Complete as of 2026-06-06.

Added `ShellSaveAdapter`, which receives a shell-owned save path and caller-provided native save document bytes, validates the document through `u3_save_open`, stages it to a unique sibling temporary file, validates staged bytes, atomically replaces the destination, and validates readback. The shell Game menu now includes Write Save Smoke, which builds a native new-game smoke document from `MainResources.rsrc` templates and writes it to the shell save path candidate.

Added `ModernShell/Tests/ShellSaveAdapterTests.swift` to exercise the actual Swift shell adapter for directory creation, write/read round trip, invalid replacement retention, destination-directory rejection, and temporary-file cleanup. Extended `harness/save_fixture_tests.c` with temp-path write/read coverage for a successful round trip and for rejecting an invalid replacement while retaining the previous valid document. Filesystem and Foundation APIs remain outside `Core/`.
