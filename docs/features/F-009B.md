# F-009B: Player Combat Attack Resolution Core

## Objective

Characterize and extract bounded player attack resolution from `CombatAttack` after direction selection has already happened.

## User Value

Player attacks are the main combat action. A portable attack-resolution slice will connect existing combat predicates and future monster damage handling while keeping input, text, sound, and animation outside the core.

## Non-Goals

- Do not port direction input, combat command polling, or the full `Combat()` loop.
- Do not port projectile drawing or hit animation as platform side effects.
- Do not port spell casting or monster turns.
- Do not change weapon or Exodus Castle behavior without documentation.

## Acceptance Notes

- Caller supplies character index, direction, weapon state, random values, and combat state.
- Preserve melee, projectile, thrown dagger, Exodus Castle weapon restriction, hit chance, damage roll, and miss behavior.
- Model text, sound, projectile path, hit animation, inventory mutation, and damage handoff as explicit results or events.
- Reuse the monster damage resolution core from `F-009A` if available.

## Affected Systems

- Legacy source files: reference only, primarily `Sources/UltimaSpellCombat.c`
- Modernization harness: likely new player attack tests
- Resources/assets: none
- Build or tooling: likely `Core/include/u3_combat.h`, `Core/src/u3_combat.c`, and `harness/Makefile`
- Documentation: feature tracker, harness README, validation strategy
