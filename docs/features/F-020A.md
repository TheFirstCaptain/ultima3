# F-020A: Shell Location Provider Smoke

## Objective

Expose the first modern shell-owned resource-root and save-document path candidates to adapters without parsing resources or writing real save data.

## User Value

The modern app must own concrete macOS locations, but resource parsing and persistence serialization must stay adapter-owned. This slice proves that handoff boundary before save/load behavior becomes risky.

## Non-Goals

- Do not implement real save serialization.
- Do not parse every bundled resource.
- Do not choose final sandbox or App Store storage policy.
- Do not add AppKit, SwiftUI, Foundation URL, or sandbox APIs to `Core/`.

## Acceptance Notes

- AppKit shell can resolve and display a bundled resource root candidate.
- AppKit shell can resolve and display a modern save-document path candidate.
- Adapter-facing handoff uses portable data such as strings, not AppKit or SwiftUI types.
- Missing paths are reported visibly in shell status text and do not crash.

## Affected Systems

- Legacy source files: reference only.
- Modern shell: location provider and shell smoke status.
- Modernization harness: likely none unless a small path-normalization helper is added outside platform code.
- Resources/assets: read-only path checks only.
- Documentation: feature doc and tracker.

## Validation Plan

Build the shell, run existing harness tests, run the Core boundary scan, and perform HIL validation that the shell reports resource and save path status.

Validation passed on 2026-06-06 with:

```bash
swift build --product Ultima3ModernShell
make -C harness test
rg -n "AppKit|SwiftUI|Metal|AVFoundation|Foundation|SDL|NS[A-Z][A-Za-z]+\\b|NSURL|Sandbox" Core
```

The boundary scan produced no matches.

Manual shell validation is still recommended by running `swift run Ultima3ModernShell` from the repository root and confirming the status overlay reports resource and save path candidates.

## Open Questions

- Resolved for this smoke slice: the shell searches repo-local `Resources` candidates for development runs and a nested `Resources` directory under `Bundle.main.resourceURL` when available.
- Deferred: the final app-bundle resource packaging contract is not established until bundle resources are explicitly configured for the modern shell.
- Deferred to F-020D: creating the save directory and performing atomic writes.

## Status

Complete as of 2026-06-06.

Added `ShellLocationProvider`, which keeps `FileManager`, `Bundle`, and concrete path ownership inside `ModernShell/Sources`. The provider reports a resource-root candidate and an Application Support save-document candidate as plain strings. `GameHostView` now displays command, resource, and save-path status lines, and the Game menu includes a Refresh Locations smoke command.
