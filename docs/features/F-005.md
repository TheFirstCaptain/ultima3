# F-005: Autocombat Movement Forecast Core

## Objective

Characterize and extract autocombat movement forecast helpers into the portable core.

## User Value

Movement forecasting is the bridge between simple target checks and the eventual full `AutoCombat` command generator. It predicts monster movement, detects future collisions, and chooses movement keys toward useful attack positions without requiring rendering, audio, resources, or spell/weapon decision branches.

## Non-Goals

- Do not port the full `AutoCombat` command generator.
- Do not port spell casting, weapon choice, healing, macro queueing, preferences, rendering, or audio.
- Do not modify legacy `Sources/`.

## Acceptance Notes

- Portable implementation covers `SetupNow`, `SetupFuture`, `FutureMonsterHere`, `DirToNearestMonster`, and `LineUpToMonster`, or namespaced equivalents.
- Harness tests cover current-position copying, future collision checks, monster movement toward the nearest character, collision fallback order, nearest-monster direction, line-up priority on left and right screen halves, and no-diagonal fallback behavior.
- Character-alive state for future collision checks is explicit in portable state.
- `make -C harness test` passes.
