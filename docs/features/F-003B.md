# F-003B: Extract Map/Math Accessors

## Objective

Move the characterized map/math accessor behavior from the harness copy into shared portable core code, then update the harness tests to call that shared implementation.

## User Value

This creates the first real portable game-core module and proves the extraction path on deterministic behavior before touching larger systems.

## Non-Goals

- Do not refactor `Sources/UltimaMisc.c`.
- Do not port map loading, saving, rendering, monsters, or resource access.
- Do not change map behavior captured in F-002B.

## Acceptance Notes

- Shared portable implementation covers `GetHeading`, `Absolute`, `GetXY`, `PutXY`, `MapConstrain`, `GetXYVal`, and `PutXYVal`, or namespaced equivalents.
- Harness tests no longer carry local copied implementations for these functions.
- The same F-002B edge cases remain covered.
- Any required state is explicit enough to avoid depending on the legacy app globals.

## Affected Systems

- Legacy source files: reference only
- Modernization harness: `harness/map_math_accessors_tests.c`, `harness/Makefile`
- Resources/assets: none
- Build or tooling: portable core source and header paths
- Documentation: validation strategy and possible architecture notes

Run:

```bash
make -C harness test
```
