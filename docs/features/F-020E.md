# F-020E: Legacy Roster Import Fixture Spike

## Objective

Add a testable path for importing a real or synthesized mutable legacy `Ultima III Roster` resource file into the modern save-domain model.

## User Value

Legacy save compatibility is valuable, but it should be proven against mutable roster data before the modern app offers import behavior to players.

## Non-Goals

- Do not promise exact legacy export.
- Do not migrate user files automatically.
- Do not write Resource Manager files.
- Do not normalize party/roster bytes into a final character model.

## Acceptance Notes

- Import path can read mutable save records for `PRTY` 400, `ROST` 400, `MAPS` 419, `MONS` 419, and `MISC` 500..505 when present.
- Missing or malformed records produce explicit import errors without partial state.
- Unknown records are retained as compatibility metadata where practical.
- Tests use a real checked-in fixture or a deterministic synthesized mutable roster fixture.

## Affected Systems

- Legacy source files: reference only, primarily roster open/read/write paths.
- Modernization harness: legacy import fixture tests.
- Resources/assets: real or synthesized mutable roster fixture.
- Build or tooling: possible fixture generator.
- Documentation: feature doc and tracker.

## Validation Plan

Run import tests for complete fixture import, missing-record failure, malformed-record failure, and preservation of uninterpreted compatibility metadata. Run full harness and Core boundary scan.

## Open Questions

- Can a real mutable `Ultima III Roster` fixture be safely checked in, or should it be synthesized from default templates?
