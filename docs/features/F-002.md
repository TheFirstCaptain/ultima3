# F-002: First Porting Harness

## Objective

Create the first executable characterization path for legacy game behavior, beginning with small deterministic C functions that can be tested without the full legacy Mac application.

## User Value

This turns the analysis baseline into a repeatable safety net. The port can begin preserving behavior before choosing or building the final modern macOS app shell.

## Non-Goals

- Do not modernize the UI.
- Do not port the full game loop.
- Do not replace Carbon, QuickTime, rendering, audio, or resource systems yet.
- Do not change historical assets.

## Acceptance Notes

- F-002A decides the initial harness/tooling approach.
- F-002B produces characterization coverage for map/math accessors.
- F-002C and F-002D are completed or explicitly deferred based on F-002B results.
- `docs/modernization/validation-strategy.md` reflects actual harness commands once they exist.
- Any durable tooling or architecture decision is recorded in `DECISION_LOG.md`.

## Result

Status: complete as of 2026-05-30.

The first executable characterization path lives under `harness/` and runs with:

```bash
make -C harness test
```

Completed child features:

- F-002A: accepted a plain C command-line harness.
