# F-053: Player Ranged Combat Attack Slice

## Objective

Support the first bounded player ranged attack paths in active combat: projectile weapons and thrown daggers.

## User Value

Players with ranged weapons should be able to use their equipped weapon identity in combat instead of seeing explicit deferred status for every non-melee path.

## Non-Goals

- Do not implement combat spells.
- Do not implement polished projectile animation or blocking pauses beyond bounded status/audio/redraw events.
- Do not implement every weapon/equipment shop or inventory management flow.
- Do not make transient combat state durable outside existing party/roster mutations.

## Dependencies

- F-050 provides the player combat command and attack-direction prompt.
- F-051 provides post-player-command monster sequencing.
- F-009B characterized bounded attack resolution and explicit projectile-hit handoff.

## Acceptance Notes

- Projectile weapon attacks compute a bounded line/path hit or miss result without platform calls from Core.
- Thrown dagger fallback mutates equipped weapon quantity and unequips when the quantity reaches the legacy empty state.
- Ranged attacks update combat state, redraw/status/audio, and then continue through the existing monster-turn sequencing.
- Inventory changes write back to the in-memory native document and mark it unsaved, while Save remains rejected in combat/dungeon mode.
- Invalid paths, unsupported weapon cases, or no target return explicit statuses without corrupting session state.
- Implemented projectile support uses the first characterized straight-line helper for weapons `3`, `5`, `9`, and `13`, and thrown dagger fallback for weapon `1` when no adjacent target is present.
- Combat presentation remains status/audio/redraw based; polished projectile animation is deferred to F-058.

## Affected Systems

- Legacy source files: `Sources/UltimaSpellCombat.c` projectile and thrown-dagger attack handling
- Portable core: combat attack/projectile path helper around existing `u3_combat_attack`
- Modern shell: ranged attack direction handling and inventory write-back
- Save adapter: active roster weapon quantity/equipment mutation
- Modernization harness and Swift tests: deterministic projectile hit/miss, thrown dagger inventory, redraw/audio/status
- Documentation: `ModernShell/README.md`, feature tracker

## Validation Plan

Cover projectile hit, projectile miss, thrown dagger quantity decrement, thrown dagger unequip, no-target status, redraw/audio feedback, document dirty marking, and save rejection while combat remains active. Run `make -C harness test`, `swift test`, and the Core boundary scan.

Manual validation: enter combat with a character whose weapon is configured for projectile or thrown dagger behavior, attack in several directions, confirm status/audio/redraw and inventory changes, and confirm Save remains rejected.
