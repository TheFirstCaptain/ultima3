# F-001A: Source Responsibility Inventory

## Objective

Inventory the responsibility of each legacy source file in `Sources/` and update `docs/modernization/legacy-inventory.md`.

## User Value

This identifies where startup, game logic, rendering, audio, resource loading, UI, and platform glue currently live.

## Non-Goals

- Do not change source code.
- Do not infer behavior beyond what file names, includes, globals, and function names support.
- Do not select a final porting technology.

## Acceptance Notes

- Each source file has a concise responsibility summary.
- Files with mixed responsibilities are called out.
- Likely subsystem ownership is grouped in the inventory table.

## Affected Systems

- Legacy source files: `Sources/*.c`, `Sources/*.m`, `Sources/*.h`
- Modernization harness: none expected
- Resources/assets: none expected
- Build or tooling: none expected
- Documentation: `docs/modernization/legacy-inventory.md`

## Validation Plan

Review the source list with `rg --files Sources` and spot-check representative files. Record inspected files and uncertainty.

## Open Questions

- Which files are pure game logic versus platform glue?
- Which files should be analyzed together because they form one subsystem?
