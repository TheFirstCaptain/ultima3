# F-004: Autocombat Targeting Core

## Objective

Characterize and extract bounded autocombat targeting helper behavior into the portable core.

## User Value

Autocombat exercises more connected gameplay rules than the first core seed while still avoiding rendering, audio, resources, and the full combat loop. This expands the portable core beyond isolated helpers into decision logic that combines combat state, map passability, monster threat, and diagonal-movement policy.

## Non-Goals

- Do not port the full `AutoCombat` command generator.
- Do not port spell casting, weapon selection, healing, rendering, audio, resource loading, or preferences.
- Do not modify legacy `Sources/`.
- Do not change behavior captured from `Sources/UltimaAutocombat.c`.

## Acceptance Notes

- Portable implementation covers `ThreatValue`, `MonsterNearby`, `MonsterLinedUp`, and `AutoMoveChar`, or namespaced equivalents.
- Harness tests cover threat scoring, poison threat doubling, cardinal and diagonal nearby priority, lined-up target direction, diagonal policy, and movement fallback/pass behavior.
- Diagonal preference is explicit state, not a direct platform preference call.
- `make -C harness test` passes.

## Result

Status: complete as of 2026-05-30.

Implemented:

- `Core/include/u3_autocombat.h`
- `Core/src/u3_autocombat.c`
- `harness/autocombat_targeting_tests.c`

The portable state uses `u3_combat_state`, future monster positions, experience values, an 11-by-11 tile array, and an explicit `allow_diagonal` flag.

## Affected Systems

- Legacy source files: reference only
- Modernization harness: `harness/autocombat_targeting_tests.c`, `harness/Makefile`
- Resources/assets: none
- Build or tooling: `Core/include/u3_autocombat.h`, `Core/src/u3_autocombat.c`
