# F-037: Location Session State and Resource Loading

## Objective

Add a bounded, in-memory location session that loads a requested non-Sosaria destination from bundled `MAPS`, `MONS`, and `TLKS` records without making read-only destination data part of the native save document.

## User Value

Transition detection only becomes useful when the shell can display and operate on the requested destination. A location session provides one explicit owner for destination map data and return-to-Sosaria context instead of spreading transient state across AppKit code.

## Non-Goals

- Do not implement movement, NPC interaction, shops, combat, or dungeon perspective rendering.
- Do not add bundled destination records to the native save format.
- Do not autosave on entry or exit.
- Do not expose Foundation or resource-file pointers through portable core headers.

## Dependencies

- F-035 provides a portable transition request with destination identity and return coordinates.
- F-013 and F-015 characterize bundled map, monster, talk, and dungeon record families.
- F-036 defines dirty-state and Sosaria-only manual-save behavior.

## Acceptance Notes

- A location-session descriptor records destination kind, location index, resource id, map shape, initial position/heading, return coordinate, and owned/caller-lifetime byte views needed by core rules.
- The resource adapter validates the required record family before replacing the current session.
- Town and castle records require compatible `MAPS`, `MONS`, and `TLKS` records; dungeon records require `MAPS` and `TLKS` and use the characterized 2048-byte dungeon shape.
- Failed or partial resource loads leave the existing Sosaria session and current save document unchanged.
- Resource-id derivation from the legacy location index is centralized and fixture-tested, including the post-index-18 offset rule.
- The shell can render a fixture-backed 2D destination frame from a successfully loaded session while F-035 remains a detection-only feature.

## Affected Systems

- Legacy source files: `Sources/UltimaMain.c`, `Sources/UltimaMisc.c`
- Portable core: a focused location/session module or explicit extensions to existing resource/render contracts
- Modern shell: `ShellResourceAdapter`, `AppDelegate`, and game-host session ownership
- Modernization harness: `resource_fixture_tests.c` plus a focused location-session test if core behavior warrants it
- Documentation: `ModernShell/README.md` and feature tracker

## Validation Plan

## Open Questions

- Resolved for this slice: `ShellResourceAdapter` copies transient destination records into immutable Swift `Data` values. The portable descriptor retains metadata and validated lengths, not pointers whose lifetime could outlive the resource buffer.
- Resolved for this slice: one generic portable descriptor validates common identity, position, return context, and record shape. Actual map bytes remain shape-specific, and later navigation APIs may still separate 2D and dungeon behavior.
- Resolved for this slice: activating a location session changes only the displayed transient frame. Commands are reported as deferred until F-038, and Refresh Locations, New Game, or Load Game restores the retained Sosaria display.
