# F-054: First Combat Spell Command Slice

## Objective

Add a first bounded player combat spell command path from active combat input through portable spell result metadata and shell mutation/status handling.

## User Value

Magic-using characters need at least one usable combat spell path so combat can move beyond weapon-only interactions while keeping the spell system testable and bounded.

## Non-Goals

- Do not implement the full cleric and wizard spell lists.
- Do not implement town, dungeon, or overworld non-combat spells.
- Do not add complex spell animation or modal blocking input loops.
- Do not change class/magic eligibility rules beyond the selected first spell subset.

## Dependencies

- F-050 provides combat command routing and non-blocking prompt patterns.
- F-051 provides monster response sequencing after handled player commands.
- F-053 may provide reusable line/target helpers if the selected first spell needs them.

## Acceptance Notes

- A combat spell command can request and consume bounded follow-up input if the selected spell needs a target or direction.
- The selected first spell subset returns explicit portable result data for success, failure, mutation, redraw, status, and audio.
- Magic/resource costs selected for the slice mutate the current in-memory native document and mark it unsaved.
- Unsupported spells, invalid casters, invalid targets, and insufficient resources report explicit status without corrupting combat state.
- Save remains rejected while combat is active.

## Affected Systems

- Legacy source files: `Sources/UltimaSpellCombat.c` combat spell command handling and selected spell helpers
- Portable core: bounded combat spell contract
- Modern shell: spell input routing and status/audio/redraw mapping
- Save adapter: active roster magic/resource mutation for the selected subset
- Modernization harness and Swift tests: deterministic spell success/failure and mutation cases
- Documentation: `ModernShell/README.md`, feature tracker

## Validation Plan

Cover valid caster, invalid caster, insufficient resource, target/direction prompt if needed, success mutation, failure/no-mutation, redraw/status/audio fields, document dirty marking, and save rejection while combat remains active. Run `make -C harness test`, `swift test`, and the Core boundary scan.

Manual validation: enter combat with a spell-capable character, invoke the selected spell command, exercise valid and invalid target paths, confirm status/audio/redraw, and confirm Save remains rejected.

## Open Questions

- Which spell is narrowest and most valuable for the first slice: a direct damage spell, an area spell, or a defensive/healing spell?
- Should spell selection use legacy letter prompts immediately, or start with one explicit smoke command before expanding the command surface?
