# F-001C: State and Coupling Inventory

## Objective

Identify major global state, shared mutable data, and coupling hotspots in the legacy source.

## User Value

This shows where behavior is hardest to extract and helps choose safe characterization targets.

## Non-Goals

- Do not refactor global state.
- Do not rename variables or move declarations.
- Do not attempt full call-graph generation unless it is lightweight and documented.

## Acceptance Notes

- Major globals and shared arrays are summarized by owning file or subsystem.
- Coupling hotspots are identified with file references.
- Risks for extraction and testing are documented.

## Affected Systems

- Legacy source files: especially `Sources/UltimaMain.c` and shared headers
- Modernization harness: none expected
- Resources/assets: none expected
- Build or tooling: none expected
- Documentation: `docs/modernization/legacy-inventory.md`

## Validation Plan

Search for global declarations, `extern` references, shared headers, and broad include files. Spot-check high-traffic files before summarizing.

## Open Questions

- Which state can become explicit core state first?
- Which globals are platform state versus game state?
