# F-009A: Monster Damage Resolution Core

## Objective

Characterize and extract `DamageMonster` behavior from `Sources/UltimaSpellCombat.c` into the portable core.

## User Value

Monster damage and defeat are central combat state transitions. Extracting this slice first gives later player attack, projectile, spell, and monster-turn work a shared, tested way to apply damage, award experience, clear defeated monsters, and restore the tile under a monster.

## Non-Goals

- Do not port the full combat loop.
- Do not port player attack targeting, projectile animation, spell casting, poison, pilfering, or monster turns.
- Do not load strings, resources, sounds, or graphics in the portable core.
- Do not change the legacy Lord British no-damage behavior unless an intentional behavior change is documented.

## Acceptance Notes

- Characterize `DamageMonster` inputs and outputs using explicit state rather than legacy globals.
- Preserve death threshold behavior, non-death HP subtraction, Lord British immunity, experience lookup, and tile restoration behavior.
- Model text, experience award, redraw, and tile-restore effects as return values or events.
- Add focused harness coverage.

## Result

Status: complete as of 2026-06-03.

`DamageMonster` is extracted as `u3_combat_damage_monster` in `Core/src/u3_combat.c`, with caller-owned state in `u3_combat_state` and observable side effects represented by `u3_combat_damage_result`. The portable function preserves ordinary HP subtraction, death on exact or overkill damage, Lord British immunity for monster type `0x26`, experience lookup through `((monster_type / 2) & 0x0F)`, defeated-monster HP clearing, and restoration of the tile under the monster.

Text output, experience award, redraw, and tile restoration are represented as result fields instead of direct legacy UI, party, or rendering calls. The result carries the legacy defeated-monster message ID (`130`), awarded experience value, target character number, redraw/restore flags, defeated coordinates, and restored tile.

## Affected Systems

## Open Questions

- Resolved for this extraction: the result carries message ID and experience value separately so adapters can decide how to present text and apply party experience.
- Resolved for this extraction: tile restoration is both a direct core tile mutation and an explicit result event so state transitions remain complete and adapters can redraw the changed tile.
